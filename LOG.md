## 2026-05-30 初版完成

### 背景与关键决策
- 原 Prompt 假设标的为期货、数据为5分钟线；但本地 `股票数据/` 实为**全A股日线（GBK编码）**，与策略不匹配。
- 与用户确认：**删除本地股票数据，坚持原期货策略**，改用 **tqsdk（天勤免费账户）** 拉取5分钟主连数据。
- 实测确认：免费账户单序列上限 **8964 根**；AG 覆盖 ~4.3 个月（含夜盘），LC/PS ~10 个月（仅日盘）。

### 已确认的规则
- 固定 1 手；同根K线止盈止损都触及时**止损优先**（保守）；时段收盘前未触发则**强平**（纯日内）。
- **双开盘都看**：夜盘开盘与日盘开盘各算一次独立开仓机会。
- 真实合约规格（tqsdk 实测，非占位符）：AG 乘数15/tick1.0，LC 乘数1/tick20.0，PS 乘数3/tick5.0。
  - 注：Prompt 占位符把 PS 乘数写成 5，实际是 3。

### 实现要点
- tqsdk `datetime` 是**纳秒UTC**，已 +8h 转北京时间；夜盘(>=20:00)归下一交易日。
- 时段切分用「相邻K线间隔>60min」自动判定，无需硬编码 09:00/21:00。
- tqsdk 仅用于一次性下载数据到 `output/cache/*.csv`，回测引擎为自写 pandas 循环，**完全离线**。

### 验证结果（端到端跑通）
- 策略逻辑单测通过；抽查 AG 一笔：入场=第3根close+1跳滑点、止损=第3根low-1跳、盈亏比精确=2.0、PnL数学吻合。
- 回测汇总（483 笔，含滑点+手续费）：胜率 ~40%、实际盈亏比 ~1.77、总收益率 ~46.9%、最大回撤 ~27.5%、夏普 ~1.16。
- 三张图（资金曲线/盈亏直方图/月度柱状）均生成，中文渲染正常。

### 已知局限
- 主连 KQ.m@ 跨合约换月有拼接跳空；历史样本偏短（4~10个月）；同根撮合用「止损优先」近似。
- 凭证通过环境变量 TQ_USER/TQ_PWD 传入，未写入代码库。

### 运行方式
```
pip install -r requirements.txt
export TQ_USER=... TQ_PWD=...   # 首次下载数据需要，之后走本地缓存
python main.py
```

## 2026-05-30 新增模拟账户（动态仓位管理）

- 新增 `backtest/account.py`：事件驱动的账户级模拟，在1手回测明细上做资金管理。
- 仓位规则（用户选定「每笔1%风险」推荐方案）：
  - 每笔手数 = floor(账户净值×1% ÷ 单手止损金额)，不足1手则跳过；
  - 同时持仓占用保证金 ≤ 账户净值×50%（保证金率按合约价值12%估算），超限削减手数；
  - 真实滚动权益：先结算到期持仓再给下一笔定手数。
- config 新增 RISK_PCT=0.01、MARGIN_CAP=0.50、MARGIN_RATE=0.12；engine 输出补 stop_price/multiplier/per_lot_risk。
- 结果对比（10万本金）：
  - 固定1手：总收益 +46.9%，最大回撤 27.5%。
  - 1%风险动态：总收益 +21.2%，**最大回撤降到 14.4%**，最差单笔从 -5,603 降到 -1,429。
  - 成交344笔/跳过139笔；AG 因单手风险大被跳40%、成交仅1-2手；LC风险小平均2.3手(最多6)。
- 产物新增：`output/account_trades.csv`、`output/account_equity.png`。

## 2026-06-01 趋势过滤（方案C：日线 vs 日内，并排对比）

- 新增 `strategy/trend.py`（日线N日均线 / 日内5分钟SMA，均防未来函数：日线shift(1)、日内截至信号bar）。
- `engine.run` 加 `trend_mode/trend_n`：信号方向需与趋势同向才开仓，否则过滤（filtered独立计数）。
- 新增 `backtest/compare.py`：三套配置对比 + N值稳健性扫描；`plot.plot_equity_compare` 叠加权益曲线。
- **结果（模拟账户口径）**：
  | 配置 | 收益% | 最大回撤% | 成交 | 过滤掉 |
  |---|---|---|---|---|
  | 无过滤(基线) | +21.2 | 14.4 | 344 | 0 |
  | 日线N20 | +12.0 | **7.2** | 158 | 265 |
  | 日内N200 | **-1.9** | 10.5 | 181 | 198 |
- **稳健性扫描**：
  - 日线 10/20/40 → 收益 +8.9/+12.0/+21.8，回撤 9.2/7.2/9.8 ——**全部正收益、回撤稳定降到7-10%，稳健不过拟合**。N40甚至保住基线收益(+21.8%)而回撤砍到9.8%（接近免费午餐）；N20回撤最低。
  - 日内 100/200/400 → 收益 +0.7/-1.9/-5.1，越大越差 ——**快均线自己也在震荡期反复翻，把好单也砍了，收益转负，弃用**。
- **结论：采用日线趋势过滤，弃用日内**。日线版把回撤砍半且收益为正、跨N稳健；日内版有害。
  - 收益/回撤取舍在日线内部：N20最稳(回撤7.2%)、N40最赚(收益21.8%/回撤9.8%)。
- 产物新增：`output/equity_compare.png`、`output/trend_compare.csv`。

## 2026-06-01 v4：RR比改为2.5，修正止盈计算

- 盈亏比从1:2改为1:2.5（config新增RR_RATIO=2.5）。
- 修正逻辑bug：止盈原来从bar3.close（未含滑点）算距离，现改为从实际入场价（含滑点）算，RR比才是真实风险回报比。
- 结果：收益23.5%，**最大回撤从8.0%降到5.0%**，期末权益123,531元。
- 盈亏比测试结论写入NOTES.md：1:1手续费吃光利润；1:4止盈太远且长期持有更合适；1:2.5最优。
- 同期指数对比：中证商品期货指数+16.7%/回撤-12.8%，我们+23.5%/回撤5.0%，跑赢6.8%且回撤小一半。

## 2026-06-01 品种扩展实验（已撤回）

- 尝试加入 RB(螺纹钢)/M(豆粕)/MA(甲醇)/I(铁矿石)，tqsdk 数据可拉（~6.6个月），合约规格已核验。
- **结果：4个新品种固定1手均亏损**（I:-12,504；MA:-3,008；M:-2,882；RB:-1,221），胜率仅28-38%。
- 动态仓位放大问题：M/MA/RB因tick=1/乘数=10止损金额极小，1%预算开出7-14手，亏损被大幅放大，账户跌至-51.8%。
- 撤回新品种，恢复 AG/LC/PS 三品种；结果验证无误（期末12.1万，+21.2%，回撤14.4%）。
- **教训：扩展品种前必须先用固定1手验证正期望，动态仓位只放大有边际的品种。**
- 有趣发现：加日线N=40趋势过滤后，7品种合计仍能 +12%/回撤18%——过滤器在帮助挡掉无效品种的坏信号，但不代表这些品种有持续正期望，不采用。
